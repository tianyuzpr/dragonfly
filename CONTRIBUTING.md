## Project Contributions

We warmly welcome you to contribute to the project. However, there are some guidelines we would like you to understand.

## Pull Request

Before creating any Pull Request, please read the following guidelines:
- Carefully review your code and ensure it complies with PEP-8 standards as much as possible.
- Include necessary documentation and comments in your Pull Request.
- Include necessary test cases in your Pull Request.

## Issue

Before creating any Issue, please read the following guidelines:
- Search existing Issues (including closed ones) to confirm that the issue has not been raised before.
- Use a clear, specific title that briefly describes the problem or suggestion.
- For bug reports, please include at least:
  - Operating system version (e.g., `uname -a`)
  - Steps to reproduce
  - Expected results vs. actual results
- For feature requests, please describe the use case and expected behavior.
- Avoid posting off-topic discussions in Issues.

## AI Coding Guidelines

If you use AI-assisted tools (such as GitHub Copilot, ChatGPT, Claude, etc.) to generate code, please note the following:
- You remain responsible for the generated code, ensuring it complies with the project's license (GPL‑v3), coding standards, and security requirements.
- You bear **full responsibility** for all code you submit, regardless of whether it was generated by AI.
- AI-generated code must be clearly marked in the Pull Request (e.g., add an `[AI-generated]` tag in the description).
- Directly copying third-party code that is not explicitly open source is strictly prohibited (AI-generated code must be manually reviewed to avoid copyright or license conflicts).
- Manually verify code logic, potential security vulnerabilities, and performance impact before submitting.
- If an AI tool suggests introducing new dependencies, evaluate whether the dependency's license is compatible with GPL‑v3.
