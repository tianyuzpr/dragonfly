# 贡献指南

[English](CONTRIBUTING.md) | 简体中文

## 项目贡献

我们非常欢迎您参与项目贡献。但是，我们有一些注意事项需要您理解。

## Pull Request

在新建任何 Pull Request 之前，请先阅读以下注意事项：
- 请仔细阅读您的代码并确保尽量符合 PEP-8 规范。
- 请在 Pull Request 中包含必要的文档和注释。
- 请在 Pull Request 中包含必要的测试用例。

## Issue

在新建任何 Issue 之前，请先阅读以下注意事项：
- 请先搜索已有的 Issue（包括已关闭的），确认该问题未被提出过。
- 请使用清晰、具体的标题，简要描述问题或建议。
- 如果是问题报告（Bug），请至少包含：
  - 操作系统版本（如 `uname -a`）
  - 复现步骤
  - 期望结果与实际结果
- 如果是功能请求（Feature Request），请说明使用场景及预期行为。
- 请避免在 Issue 中发表与主题无关的讨论。

## AI Coding 注意事项

如果您使用 AI 辅助工具（如 GitHub Copilot、ChatGPT、Claude 等）生成代码，请注意以下几点：
- 您仍需对生成的代码负责，确保其符合项目的许可证（GPL‑v3）、编码规范及安全要求。
- 所有您提交的代码，无论是否由 AI 生成，您均必须对代码负**全部责任**。
- AI 生成的代码必须在 Pull Request 中明确标注（例如在描述中添加 `[AI-generated]` 标记）。
- 严禁直接复制未明确开源的第三方代码（AI 生成的代码需经过人工审查，避免版权或许可证冲突）。
- 提交前请人工验证代码逻辑、潜在安全漏洞及性能影响。
- 如果 AI 工具建议引入新依赖，请评估该依赖的许可证是否与 GPL‑v3 兼容。
